
# C Master Path 

> A deep systems-engineering journey into the C programming language.
> 365 days of disciplined study, practice, and low-level mastery.

---

## 🎯 Mission

This repository documents my structured journey to mastering C at a systems level.

The goal is not just to "learn C", but to understand:

- Memory at a deep level
- Pointer mechanics
- Dynamic allocation
- Modular architecture
- Compilation & linking process
- Debugging and memory analysis
- Writing clean and reusable C libraries

---

## 🗺 Roadmap

### 🔹 Phase 1 — Foundations (Day 1–90)
- Syntax and types
- Control flow
- Functions
- Arrays & Strings
- Structs

### 🔹 Phase 2 — Memory & Pointers (Day 91–210)
- Pointer mastery
- Dynamic memory (malloc, free)
- Linked lists
- Stack vs Heap
- Memory layout understanding

### 🔹 Phase 3 — Professional Structure (Day 211–300)
- Header files
- Modular design
- Static libraries
- Makefiles
- GDB debugging
- Valgrind memory analysis

### 🔹 Phase 4 — Consolidation (Day 301–365)
- Larger CLI systems
- Refactoring
- Rewriting core data structures from scratch
